> 最后更新: 2026-07-08

## OPC 定位

**"微信 + Soul" 为 OPC 主理人服务，80% ToB，20% ToC。**

一人公司（One Person Company）经营者的操作系统——用社交连接同行、用知识库沉淀资产、用 AI Agent 辅助决策、用 Skill 标准化服务、用交易市场变现。

## 二、逐竞品深度分析

### 1. 微信

**体量**：12亿 MAU，国民级即时通讯基础设施。

**不可替代之处**：IM 本身。用户之间聊合作、加好友，最终还是会交换微信。OPC 不应该做 IM——做不过微信，也没有必要。

**OPC 差异化**：

- **朋友圈是封闭的**（只能看好友），OPC Feed 可以做到**同行业/同技能/同地域的公开内容发现**
- 微信没有知识库、没有 Agent、没有设备管理——这些都是 OPC 的独占价值
- 微信的"一人公司"经营场景完全空白——没有专门的工具链

**策略**：不与微信正面竞争。OPC 做"微信之上的专业层"——微信聊天，OPC 经营。

---

### 2. 飞书

**体量**：3000万 MAU，字节跳动旗下，企业协作第一梯队。

**不可替代之处**：飞书是给**有组织的企业**用的——OKR、审批流、日历、视频会议。一人公司不需要这些重型组织工具。

**OPC 差异化**：

- 飞书知识库是**企业内部私有文档**，OPC 知识库是**可公开、可定价、可交易的工业知识资产**
- 飞书 Aily 智能体是**企业级工作流自动化**，OPC Agent 是**个人经营的 AI 顾问**——定位不同
- 飞书没有社交层，没有交易层，没有设备运维

**关键差异**：飞书服务的是一家公司**内部的协作效率**，OPC 服务的是一人公司**之间的连接和交易**。不是一个赛道。

---

### 3. 企业微信

**体量**：1亿 MAU，1400万+企业，腾讯旗下。

**最接近 OPC 的竞品**。企业微信的核心是"微信私域经营工具"——加客户微信、发朋友圈、群发消息。OPC 的主理人也需要经营客户关系。

**不可替代之处**：和 12 亿微信用户互通。这是腾讯的护城河，OPC 不可能也没有必要去挑战。

**OPC 差异化**：

- 企业微信没有**内容社区**——你不会在企微里逛 Feed
- 企业微信没有**知识库 / AI Agent / 设备运维**
- 企业微信没有**交易市场**（商品上架、订单管理）
- 企业微信的 AI 能力依赖第三方 SCRM 插件，OPC 是原生 Agent

**策略**：企业微信是"对外经营客户"的工具，OPC 是"对内经营自己"的工具。两者互补而非替代。

---

### 4. 百度贴吧

**体量**：8000万 DAU，中国最大的兴趣社区（23年历史）。

**最接近 OPC 社交层的竞品**。贴吧是"兴趣吧"，OPC 的圈子按行业/技能分组，结构类似。

**不可替代之处**：贴吧有 23 年的内容积累、8000万日活、10万个兴趣吧。OPC 冷启动不可能复制这个规模。

**OPC 差异化**：

- 贴吧是**匿名/昵称制**，OPC 是**实名/技能名片制**——贴吧的信任成本高，OPC 的信任机制更适合商业合作
- 贴吧是纯内容社区，没有**工具链**（知识库/Agent/设备/交易）
- 贴吧用户是**兴趣驱动**（游戏、娱乐），OPC 用户是**职业驱动**（运维、自动化、硬件）——质量密度不同
- 贴吧 2026 版偏向游戏/娱乐垂类，**工业/运维垂类完全空白**

**策略**：OPC 的圈子模型直接借鉴贴吧的"吧"结构——但不是做"兴趣吧"，是做"技能圈"。每个圈子内嵌工具链（知识库/Agent/Skill），而非纯灌水。

---

### 5. 闲鱼 / 淘宝

**体量**：闲鱼 5亿+ 用户，中国最大的 C2C 二手交易平台。

**OPC 交易层对标**。闲鱼的"技能服务"板块（修手机、做设计、写代码）和 OPC 的 trade 模块有交集。

**不可替代之处**：闲鱼有完整的支付、物流、信用体系、纠纷仲裁——这是大型电商基建，OPC 短期内不可能也不需要重建。

**OPC 差异化**：

- 闲鱼交易是**一次性的**（买完即走），OPC 交易发生在**社交关系之上**（先关注→先交流→再交易→交易完还能看他后续的知识库更新）
- 闲鱼没有**知识资产交易**——OPC 的"知识库文档可定价"是独有的内容变现模式
- 闲鱼没有**服务标准化**——OPC 的 Skill 系统可以把一个人的能力打包成可执行、可复用的标准化服务

**策略**：OPC 不做实物电商，做**服务和知识的交易**。不接入支付和物流，把商品模块定位于"服务上架→订单记录→交付确认"的轻量流程。

## 五、冷启动路径（对标贴吧的"吧"模型）

贴吧活 23 年的核心：**垂直吧**。不是因为"贴吧"这个品牌强，是每一个具体的话题吧有自己的活力和吧主。

OPC 同理——不是做一个"大平台"让人来，是做**垂直技能圈**：

| 圈子 | 内容方向 |
|------|---------|
| **OPC 运维圈** | 服务器/网络/自动化/监控经验分享 |
| **OPC 硬件圈** | Arduino/树莓派/工控机/NAS 交流 |
| **OPC 自动化圈** | Python 脚本/Home Assistant/PLC 编程 |
| **OPC 一人公司圈** | 税务/法务/接单/定价/客户管理 |

每个圈子有独立的 Feed、知识库、Skill 列表。圈子之间的人和内容可以互通。
